# FFTLivingWeapons -- Staff + Sword Live Test Plan
*Work top to bottom; blockers surface first. Check each box live. Created 2026-06-16, for the Choir release.*

**Scope.** Deep coverage of the four STAFF signatures (Benediction, Renewal, Sanctuary, Choir) and the
SWORD line (items + signatures + base mechanics); everything else is a regression pass. Cross-cutting and
cross-signature crazy edge cases follow. This is the deferred "comprehensive staff-set test plan" plus the
sword pass, written now that all four staves are in (Choir shipped 2026-06-16, commit bf66628).

**Gating model.** Living-weapon +3 signatures fire only at kill-tier 3. On a DEV build (`BuildLinked.ps1`)
every weapon is seeded to +3 instantly, so verify the GATE and the EFFECT here. On a PROD build
(`Publish.ps1` / `BuildLinked.ps1 -Prod`) nothing is seeded and the curve is {5,25,50}, so verify the
CURVE (no premature firing; 50 kills to +3). Many checks below are trivial on DEV and a grind on PROD --
do the mechanic pass on DEV, then a pacing/no-premature-fire spot check on PROD before release.

**Severity.** [BLOCKER] = do not release. [MAJOR] = fix before release. [MINOR] = log it. [EDGE] = crazy
corner case, verify if reachable in a real battle.
